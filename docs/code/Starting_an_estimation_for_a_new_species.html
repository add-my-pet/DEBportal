Download the DEBtool[edit]

For Matlab, download DEBTool_M.zip file here, unzip it and set the path to the DEBtool_M directory and all its subdirectories. Always work from your own directory outside of the DEBtool folder. Start the AmP entry preparation system by typing AmPeps

Think about which typified model(s) is/are applicable to your species[edit]

A number of extensions on the standard DEB model are needed to capture specific life-history of various groups of species - these extensions on the standard DEB model are called typified models. Each one is attributed its own label. This has some importance for the procedure.

Some examples of model extensions are relaxing the assumption of constant shape whereby changes in shape allow metabolism to accelerate for part of the life-cycle (Kooijman, Pecquerie et al. 2011). Or including extra-life stages (for instance weaning for mammals, pupation in insects), and choosing different modes of reproduction (foetal or egg development). Parameter values can still be compared over across different species.

Go the AmP species list page and see what Typified model corresponds to a similar species or taxonomic group. You will see that almost all species of a given class or order have the same model, which is given by DEBtool function get_model. Bear in mind, that this is a starting point and you may need some research and testing to come to a final decision on which Typified model applies to the species.

Create a folder which will contain the 4 user-defined files of your species[edit]

Create a repository with the name of 'your pet' e.g. Chromis_chromis.

You will need the following four files in your working folder:

  • mydata_Chromis_chromis: for setting data, metaData, and references;
  • predict_Chromis_chromis: for specifying model predictions;
  • pars_init_Chromis_chromis: for initiating parameter values and fix/free them: and finally
  • run_Chromis_chromis: for checking, setting estimation options and running the estimation


The easiest way to get them there is to use the AmP entry prepare system. Just type AmPeps and have a look in the manual that will load in your browser. You need to be connected to the internet.

Alternatively, but more risky, download the files of a similar species online and modify them, not forgetting to edit all data. One way to be sure of that, start by deleting all bibkeys in the mydata-file. If your species is already in the collection, download the 4 files, add your name in the mydata-files as modification author, and a discussion point that explains what changed and why.

Make the entry[edit]

After having generated the four source files

  • mydata file
  • pars_init file
  • predict file
  • run file

using AmPeps, carefully check all fields to see if they need further editing. Have a look at existing files in the collection for inspiration, especially if you have advanced data types, that we have not implemented in the templates used by AmPeps. If AmPeps could not find code for a particular data set, it will mention so in the predict file, and you have to copy the copy from some other predict file, or code it yourself. The page Species list shows the data types for each species.

Depending on option settings in the run file, results will be printed to an html file and/or a .mat file. This .mat file can be read with printmat('my_pet'). You can start parameter estimation from an automatized setting, the initial setting as specified in pars_init, or from the results_my_pet.mat file, which was written after a previous run. You can reduce or enhance the effect of particular data sets/points by changing weight coefficients in mydata. The function parscomp_st is used in predict files to compute compound parameter from primary parameters; most DEBtool/animal functions work with scaled parameters. The script run_my_pet first checks the consistency of the three user-defined functions with check_my_pet, assuming that they have no Matlab errors. Then estimation-options are set, such as the maximum number of iterations, or no estimation at all. Finally the estimation and automatized presentation of results are called; the presentation of results can (optionally) be customized with function custom_results_my_pet. Plots with the same labels and units can be combined into one plot, by assigning datasets to a group and setting a caption.

Start estimation[edit]

Before you start parameter estimation, it makes sense to study the effect of parameter values on the predictions. This is done using, in run_my_pet, the two options:

  • estim_options ('method', 'no')
  • estim_options('pars_init_method', 2)

If the predictions are really far off, you can edit the parameter values, following the nine steps procedure. The estimation procedure is usually fairly forgiving, so you only do this if the predictions are really far off (meaning a factor 10 or 100, don't care for a factor 2). If the predictions are not too far off, you can start estimation, by setting 'method' on 'nm' (= Nead Melder, a simplex method). The 'max_step_number' is preferable not set at a large number, 500 will do. The typical way to proceed is first use 'pars_init_method' 2 to start from the values specified in the pars_init file, then choose 1 to continue with the values that were written in the .mat file. Repeat till satisfaction (arrow-up + enter), then type mat2pars_init to copy the values in the .mat file to the pars_init file.
